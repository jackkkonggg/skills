# TypeScript Clean Code

Apply Robert C. Martin's Clean Code principles, adapted for idiomatic TypeScript, when writing, reviewing, or auditing TypeScript codebases.

## Enforcement policy

- Use recommendation-first language for style, naming, and organizational guidance.
- Use strict language (`must` / `must not`) only for type safety, error correctness, or runtime failure prevention.
- Prefer the official TypeScript documentation and handbook when resolving conflicts; update these rules when best practices evolve.
